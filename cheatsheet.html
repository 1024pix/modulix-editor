[EN COURS DE CONSTRUCTION ET DEV] Modulix — JSON Schema Cheat Sheet

Reference complete de tous les champs, types et proprietes du schema de module Pix. Genere le 13 mars 2026

Structure Module Sections Grains Components Elements Custom QROCM
Structure generale
Module
  ├── id, shortId, slug, title, isBeta, visibility
  ├── details { image, description, duration, level, objectives, tabletSupport }
  └── sections[]
        ├── id, type
        └── grains[]
              ├── id, type, title
              └── components[] (oneOf)
                    ├── { type: "element", element: Element }
                    └── { type: "stepper", steps: [{ elements: Element[] }] }
Module
Proprietes du module au niveau racine.
Legende : * = requis stringbooleanintegerarrayobject format
ProprieteTypeValeurs / FormatDescription
id*stringuuidIdentifiant universel unique (uuid) du module.
shortId*stringIdentifiant court unique du module, présent dans l'url.
slug*string^[a-z0-9-]+$Identifiant texte unique du module, présent dans l'url. Caractères autorisés : Tout caractère entre a et z (minuscules), tout chiffre (0 à 9) et le trait d'union (-).
title*stringTitre du module.
isBeta*boolean
visibility*string
privatepublic
Valeurs acceptées : private, public. Si vous indiquez "public", le module pourra être sélectionné à la création d’un contenu formatif dans Pix Admin.

details.*

ProprieteTypeValeurs / FormatDescription

description*stringjoditTexte d’introduction en dessous du titre du module.

level*string
noviceindependentadvancedexpert
Niveau du module.
objectives*array de stringUn objectif minimum. Ils s’affichent dans l’ordre contribué.
tabletSupport*string

Si la valeur est inconvenient ou obstructed, on indiquera à l'utilisateur que le module peut être difficile à réaliser sur un petit écran.

QROCM — Detail des proposals
Les proposals d'un QROCM sont une sequence de blocs text, input et select (oneOf). Ils s'affichent les uns a la suite des autres pour former la question.
text
type*text
contentstringjodit
input
type*input
input*stringIdentifiant unique obligatoire (non visible dans le module)
inputType*string
textnumber
Le type number affiche un champ qui n'accepte que les chiffres. Le type text affiche un champ textuel classique.
size*number≥ 1Largeur du champ. Indiquez une valeur correspondant au nombre de caractères attendu.
display*string
inlineblock
Type d'affichage du champ. En inline, le champ apparaîtra sur la même ligne que les autres propositions. En block, il se mettra à la ligne suivante.
placeholder*stringTexte de substitution qui s'affiche dans le champ avant qu'il soit édité.
ariaLabel*stringDescription du champ nécessaire à l’accessibilité (non visible dans le module, lu par les lecteurs d'écran).
tolerances*array de stringLes tolérances permettent de valider une réponse malgré les erreurs. (T1 - Espaces, casse & accents, T2 - Ponctuation et T3 - Distance d'édition).
solutions*array de ? | ?Solution(s) du champ.
select
type*select
input*stringIdentifiant unique obligatoire (non visible dans le module)
display*string
inlineblock
Type d'affichage du champ. En inline, le champ apparaîtra sur la même ligne que les autres propositions. En block, il se mettra à la ligne suivante.
placeholder*stringTexte de substitution qui s'affiche dans le champ lorsqu’aucune option n'est sélectionnée.
ariaLabel*stringDescription du champ nécessaire à l’accessibilité (non visible dans le module, lu par les lecteurs d'écran).
tolerances*arrayLes tolérances ne concernent que les QROCm de type input.
options*array de { id, content }Options du champ.
solutions*array de stringSolution(s) du champ.
